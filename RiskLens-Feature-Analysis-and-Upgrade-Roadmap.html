RiskLens – Feature Analysis & Upgrade Roadmap

Purpose: Status of six features (scenario comparison, sensitivity, recommendation engine, executive summary/report, optimization, ABM) and what to upgrade for interview-ready demos.

1. Scenario comparison

AspectStatus
ExistsFully
WhereCompare page, shared comparison helpers
Filessrc/app/scenarios/compare/page.tsx, src/lib/scenarioComparison.ts, src/app/dashboard/page.tsx

What’s there: Select up to 6 scenarios; summary cards per scenario; single comparison table (rows = metrics, columns = scenarios); break-even from stored inputs; dashboard link and hint.

Interview-ready upgrades: Add sensitivity comparison (e.g. top driver per scenario or “sensitivity on detail” in card); optional sort/export of comparison table; short narrative above table (e.g. “Base Case has highest expected profit”).

2. Sensitivity analysis

AspectStatus
ExistsFully
WhereAnalytics lib + scenario detail page + TornadoChart
Filessrc/lib/analytics.ts, src/components/TornadoChart.tsx, src/app/scenarios/[id]/page.tsx

What’s there: One-at-a-time ±10% for fixed cost, price, demand mean/std, variable cost; bars ranked by impact; TornadoChart; runs after simulation; recommendation uses top 2 drivers.

Interview-ready upgrades: Configurable shock (±5%, ±10%, ±20%); document method in UI; optional two-way sensitivity; clear loading/error state.

3. Analyst recommendation engine

AspectStatus
ExistsFully (rule-based)
WhereAnalytics + RecommendationBox on scenario detail
Filessrc/lib/analytics.ts, src/components/RecommendationBox.tsx, src/app/scenarios/[id]/page.tsx

What’s there: Rule-based bullets (profit vs risk bands, P(loss), break-even, percentiles, top 2 sensitivity drivers) in “Analyst recommendation” box.

Interview-ready upgrades: Name it “rule-based recommendation engine”; add 1–2 actionable next steps when risk is high; optional comparison-level recommendation; unit tests for buildRecommendationText.

4. Executive summary / report generator

AspectStatus
ExistsNot at all
WhereN/A
FilesNone. InterpretationBox exists but is unused.

Interview-ready upgrades (from scratch): Add executive summary block (2–4 sentences: expected profit, P(loss), break-even, main drivers). Add report generator: compose scenario name, assumptions, metrics, sensitivity summary, recommendation; output HTML or “Print / Save as PDF” from browser. Optional: compare report. Use or remove InterpretationBox.

5. Optimization engine

AspectStatus
ExistsNot at all
WhereN/A
FilesNone. Break-even is analytical only; no optimizer.

Interview-ready upgrades (from scratch): Single-variable optimization (e.g. selling price to maximize expected profit) via grid search over runMonteCarlo; one section on scenario detail (“Suggested price”); document as “single-variable optimization; extendable to multi-variable or solver.”

6. ABM module

AspectStatus
ExistsNot at all
WhereN/A
FilesNone. No agents or agent-based logic.

Interview-ready upgrades (from scratch): Option A: lightweight “ABM” (e.g. demand as N agents with simple buy rule); one extra mode alongside Normal/Triangular/Uniform. Option B: “ABM module (planned)” in docs/UI and describe design in interview.

Suggested order of work (for interviews)

  1. Executive summary + report generator – High impact, reuses metrics and recommendation.
  2. Recommendation engine polish – Naming, actionable bullets, optional compare recommendation.
  3. Sensitivity – Configurable ±% and one-sentence methodology.
  4. Scenario comparison – Narrative and optional export/sort (and sensitivity in compare if time).
  5. Optimization – Single-lever grid search + “Suggested price” on scenario detail.
  6. ABM – Minimal agent-based demand mode or “planned” note and design.
